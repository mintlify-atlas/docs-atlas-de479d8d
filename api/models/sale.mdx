## Overview

The Sale (Venta) model represents a completed transaction in the Luis IT Repair POS system. It captures the full transaction details including items sold, payment methods, tax calculations, and loyalty point redemption/earning.

## Firestore Collection

**Collection Name:** `ventas`

## Core Fields

<ResponseField name="id" type="string">
  Sale document ID, automatically generated by Firestore.
</ResponseField>

<ResponseField name="fecha" type="timestamp" required>
  Transaction date and time.
</ResponseField>

<ResponseField name="clienteTelefono" type="string">
  Customer phone number if provided. Links to customer loyalty account.
</ResponseField>

## Financial Details

<ResponseField name="subtotal" type="number" required>
  Sum of all items before discounts and tax.
  
  **Calculation:** `Σ (producto.precioVenta * producto.cantidad)`
</ResponseField>

<ResponseField name="descuentoManual" type="number" default={0}>
  Manual discount amount applied by cashier.
</ResponseField>

<ResponseField name="descuentoPuntos" type="number" default={0}>
  Discount from redeemed loyalty points. Typically 1 point = 1 peso.
</ResponseField>

<ResponseField name="aplicarIVA" type="boolean" required>
  Whether sales tax (IVA) was applied to this transaction.
  
  **Note:** Configurable per transaction based on POS settings.
</ResponseField>

<ResponseField name="ivaPorcentaje" type="number">
  Tax rate applied (e.g., `0.16` for 16% IVA).
</ResponseField>

<ResponseField name="iva" type="number">
  Calculated tax amount.
  
  **Calculation:** `(subtotal - descuentoManual - descuentoPuntos) * ivaPorcentaje`
</ResponseField>

<ResponseField name="total" type="number" required>
  Final transaction amount.
  
  **Calculation:** `subtotal - descuentoManual - descuentoPuntos + iva`
</ResponseField>

## Payment Information

<ResponseField name="tipoPago" type="string" required>
  Primary payment method.
  
  **Possible values:**
  - `efectivo` - Cash
  - `tarjeta` - Credit/debit card
  - `transferencia` - Bank transfer
  - `mixto` - Multiple payment methods
</ResponseField>

<ResponseField name="pagoDetalle" type="object">
  Detailed breakdown of payment by method. Used for end-of-day reconciliation.
  
  <Expandable title="Payment Details">
    <ResponseField name="efectivo" type="number" default={0}>
      Cash amount received
    </ResponseField>
    
    <ResponseField name="tarjeta" type="number" default={0}>
      Card payment amount
    </ResponseField>
    
    <ResponseField name="transferencia" type="number" default={0}>
      Transfer amount
    </ResponseField>
    
    <ResponseField name="referenciaTarjeta" type="string">
      Card authorization or transfer reference number
    </ResponseField>
  </Expandable>
</ResponseField>

## Loyalty Program

<ResponseField name="puntosGenerados" type="number" default={0}>
  Loyalty points earned from this transaction.
  
  **Calculation Logic:**
  - Only products with `generaPuntos: true` contribute
  - Typically 1 point per peso spent
  - Points awarded to customer account after sale completion
</ResponseField>

## Items Sold

<ResponseField name="productos" type="array" required>
  Array of items included in the transaction. Each item contains:
  
  <Expandable title="Product Item Structure">
    <ResponseField name="id" type="string">
      Product document ID from `productos` collection
    </ResponseField>
    
    <ResponseField name="nombre" type="string">
      Product name (snapshot at time of sale)
    </ResponseField>
    
    <ResponseField name="codigo" type="string">
      Product barcode/SKU
    </ResponseField>
    
    <ResponseField name="precioVenta" type="number">
      Unit price (snapshot at time of sale)
    </ResponseField>
    
    <ResponseField name="cantidad" type="number">
      Quantity sold
    </ResponseField>
    
    <ResponseField name="generaPuntos" type="boolean">
      Whether this item contributes to loyalty points
    </ResponseField>
    
    <ResponseField name="esServicio" type="boolean">
      True if this is a completed service ticket being paid
    </ResponseField>
    
    <ResponseField name="folioServicio" type="string">
      Service ticket folio if `esServicio: true`
    </ResponseField>
    
    <ResponseField name="servicioId" type="string">
      Service ticket document ID if `esServicio: true`
    </ResponseField>
  </Expandable>
</ResponseField>

## Related Functions

- `registrarVenta()` - Create new sale record
- `descontarStock()` - Update product inventory after sale
- `sumarPuntosCliente()` - Award/redeem loyalty points
- `imprimirTicketVenta()` - Generate receipt

See [POS Firebase](/api/services/pos) for detailed function documentation.

## Transaction Flow

1. **Item Selection** - Build cart with products/services
2. **Client Lookup** - Optional phone number for loyalty
3. **Discount Application** - Manual + points redemption
4. **Tax Calculation** - Apply IVA if enabled
5. **Payment Processing** - Record method(s) and amounts
6. **Inventory Update** - Deduct stock for sold products
7. **Loyalty Update** - Award/redeem points
8. **Service Update** - Mark services as `entregado` and `cobradoEnPOS`
9. **Receipt Generation** - Print transaction details

## End-of-Day Reconciliation

Sales data is used in cash register close-out reports (`cortes_caja`):

```javascript
import { cerrarCajaHoy } from './js/services/corte_caja_firestore';

// Close register and generate summary
const ventasHoy = await obtenerVentasPorFecha(fechaHoy);
const corte = await cerrarCajaHoy(ventasHoy, {
  efectivoContado: 5230,
  notasEmpleado: "Todo en orden"
});

// corte includes:
// - Total sales by payment method
// - Expected vs actual cash
// - Number of transactions
// - Discrepancy alerts
```

See [Cash Register Operations](/guides/cash-register-operations) for daily closing procedures.

## Analytics & Reports

Sales records enable business intelligence queries:

- **Daily Revenue:** Sum `total` grouped by date
- **Payment Mix:** Aggregate `pagoDetalle` amounts
- **Top Products:** Count/sum by product ID
- **Customer Frequency:** Count sales by `clienteTelefono`
- **Tax Reporting:** Sum `iva` for tax filings
- **Discount Analysis:** Track `descuentoManual` and `descuentoPuntos`

Implement using Firestore queries or export to analytics tools.
