## Installation Overview

</Warning>

Clone the Luis IT Repair source code:

## Part 2: Firebase Project Setup

### Create Firebase Project

<Steps>
  <Step title="Access Firebase Console">
    Navigate to [Firebase Console](https://console.firebase.google.com/)
    
    Sign in with your Google account
  </Step>
  
  <Step title="Create New Project">
    Click **Add project**
    
    - **Project name**: Enter a descriptive name (e.g., "luis-it-repair-prod")
    - **Google Analytics**: Optional (recommended for production)
    - **Analytics account**: Select existing or create new
    
    Click **Create project** and wait for initialization
  </Step>
  
  <Step title="Configure Project Settings">
    After creation, go to **Project Settings** (gear icon)
    
    Note your **Project ID** - you'll need this for deployment
  </Step>
</Steps>

## Part 3: Firebase Services Configuration

### Enable Firebase Authentication

<Steps>
  <Step title="Navigate to Authentication">
    In Firebase Console, go to **Build** → **Authentication**
    
    Click **Get started**
  </Step>
  
  <Step title="Enable Email/Password Provider">
    Go to **Sign-in method** tab
    
    Find **Email/Password** in the native providers list
    
    Toggle **Enable** and click **Save**
  </Step>
  
  <Step title="Configure Password Policy (Optional)">
    Go to **Settings** tab
    
    Adjust password requirements:
    - Minimum length (default: 6)
    - Require uppercase/lowercase
    - Require numbers/symbols
  </Step>
</Steps>

### Initialize Cloud Firestore

<Steps>
  <Step title="Create Firestore Database">
    Navigate to **Build** → **Firestore Database**
    
    Click **Create database**
  </Step>
  
  <Step title="Select Database Mode">
    Choose **Start in production mode**
    
    We'll deploy proper security rules in the next step
  </Step>
  
  <Step title="Choose Database Location">
    Select a region close to your users:
    
    - **North America**: `us-central1`, `us-east1`
    - **South America**: `southamerica-east1`
    - **Europe**: `europe-west1`
    - **Asia**: `asia-northeast1`
    
    <Note>Database location cannot be changed after creation.</Note>
  </Step>
  
  <Step title="Complete Setup">
    Click **Enable** and wait for database initialization
  </Step>
</Steps>

### Deploy Firestore Security Rules

<Steps>
  <Step title="Install Firebase CLI">
    Install the Firebase command-line tools globally:
    
    ```bash
    npm install -g firebase-tools
    ```
    
    Verify installation:
    ```bash
    firebase --version  # 13.0.0 or higher
    ```
  </Step>
  
  <Step title="Login to Firebase">
    Authenticate with your Google account:
    
    ```bash
    firebase login
    ```
    
    This will open a browser window for authentication
  </Step>
  
  <Step title="Link Project">
    Initialize Firebase in your project directory:
    
    ```bash
    firebase use --add
    ```
    
    Select your Firebase project from the list
    
    Enter an alias (e.g., "default" or "production")
  </Step>
  
  <Step title="Review Security Rules">
    The included `firestore.rules` file provides:
    
    ```rules firestore.rules
    rules_version='2'
    
    service cloud.firestore {
      match /databases/{database}/documents {
        
        // Public read for service status (customer portal)
        match /servicios/{document=**} {
          allow read: if true;
          allow write: if request.auth != null;
        }
        
        // Authenticated users only
        match /autorizados/{document=**} {
          allow read, write: if request.auth != null;
        }
        
        match /clientes/{document=**} {
          allow read, write: if request.auth != null;
        }
        
        match /productos/{document=**} {
          allow read, write: if request.auth != null;
        }
        
        match /ventas/{document=**} {
          allow read, write: if request.auth != null;
        }
        
        match /cortes_caja/{document=**} {
          allow read, write: if request.auth != null;
        }
        
        match /folios/{document=**} {
          allow read, write: if request.auth != null;
        }
        
        match /contadores_folio/{document=**} {
          allow read, write: if request.auth != null;
        }
        
        match /egresos_diarios/{document=**} {
          allow read, write: if request.auth != null;
        }
      }
    }
    ```
  </Step>
  
  <Step title="Deploy Rules">
    Deploy security rules to Firebase:
    
    ```bash
    firebase deploy --only firestore:rules
    ```
    
    Output:
    ```
    === Deploying to 'your-project-id'...
    
    i  firestore: checking firestore.rules for compilation errors...
    ✔  firestore: rules file firestore.rules compiled successfully
    i  firestore: uploading rules firestore.rules...
    ✔  firestore: released rules firestore.rules
    
    ✔  Deploy complete!
    ```
  </Step>
</Steps>

### Configure Cloud Functions (Optional)

<Steps>
  <Step title="Install Functions Dependencies">
    Navigate to the functions directory:
    
    ```bash
    cd functions
    npm install
    cd ..
    ```
  </Step>
  
  <Step title="Review Function Code">

    ```typescript functions/src/index.ts
    import * as functions from "firebase-functions";
    
    export const mlSearch = functions
      .region("southamerica-east1")

        // Proxy for Mercado Libre API
        // Used by price comparison feature
      });
    ```
  </Step>
  
  <Step title="Deploy Functions">
    Deploy Cloud Functions:
    
    ```bash
    firebase deploy --only functions
    ```
    
    <Note>Functions require Firebase Blaze (pay-as-you-go) plan.</Note>
  </Step>
</Steps>

## Part 4: Database Initialization

### Create Initial Collections

<Steps>
  <Step title="Navigate to Firestore Console">
    Go to **Firestore Database** in Firebase Console
  </Step>
  
  <Step title="Create Folio Counter">
    Click **Start collection**
    
    - Collection ID: `contadores_folio`
    - Document ID: `global`
    - Fields:
      ```
      servicios: 0 (number)
      ventas: 0 (number)
      ```
    
    This tracks the next available folio numbers
  </Step>
  
  <Step title="Create Collections Structure">
    The application will auto-create collections on first use:
    
    - `servicios` - Service orders
    - `clientes` - Customers
    - `productos` - Product inventory
    - `ventas` - Sales transactions
    - `cortes_caja` - Daily cash reports
    - `egresos_diarios` - Daily expenses
    - `empleados` - Employee records
    - `autorizados` - Authorized users
  </Step>
</Steps>

### Create Admin User

<Steps>
  <Step title="Add User via Authentication">
    Go to **Authentication** → **Users**
    
    Click **Add user**
    
    - **Email**: admin@yourcompany.com
    - **Password**: Choose a strong password
    
    Click **Add user** and copy the generated **User UID**
  </Step>
  
  <Step title="Authorize in Firestore">
    Go to **Firestore Database**
    
    Create document in `autorizados` collection:
    
    - Collection ID: `autorizados`
    - Document ID: Paste the User UID from previous step
    - Fields:
      ```
      activo: true (boolean)
      email: "admin@yourcompany.com" (string)
      nombre: "Administrator" (string)
      online: false (boolean)
      rol: "admin" (string)
      permisos: {
        servicios.crear: true,
        servicios.ver: true,
        servicios.editar: true,
        clientes.ver: true,
        clientes.editar: true,
        productos.ver: true,
        productos.editar: true,
        ventas.pos: true,
        reportes.ver: true,
        configuracion.ver: true,
        empleados.gestionar: true
      } (map)
      ```
  </Step>
</Steps>

<Warning>
  The user must exist in both **Authentication** and **autorizados** collection with matching UIDs.
</Warning>

## Part 7: Deploy to Firebase Hosting

### Configure Firebase Hosting

The `firebase.json` configuration is already set up:

```json firebase.json
{
  "hosting": {
    "public": "dist",
    "ignore": [
      "firebase.json",
      "**/.*",
      "**/node_modules/**"
    ],
    "rewrites": [
      {
        "source": "/api/ml/search",
        "function": {
          "functionId": "mlSearch",
          "region": "southamerica-east1"
        }
      },
      {
        "source": "**",
        "destination": "/index.html"
      }
    ]
  }
}
```

<Note>
  The rewrite rules enable:
  - SPA routing (all routes serve index.html)
  - Cloud Functions integration for API proxying
</Note>

### Deploy to Hosting

<Steps>
  <Step title="Build Application">
    Ensure you have a fresh production build:
    
    ```bash
    npm run build
    ```
  </Step>
  
  <Step title="Deploy to Firebase">
    Deploy everything (hosting + rules + functions):
    
    ```bash
    firebase deploy
    ```
    
    Or deploy only hosting:
    

## Troubleshooting

<AccordionGroup>
  <Accordion title="Build fails with memory errors">
    Increase Node.js memory limit:
    
    ```json package.json
    {
      "scripts": {
        "build": "NODE_OPTIONS=--max_old_space_size=4096 vite build"
      }
    }
    ```
  </Accordion>
  
  <Accordion title="Firebase deploy fails with permissions error">
    Ensure you're logged in to the correct account:
    
    ```bash
    firebase logout
    firebase login
    firebase use your-project-id
    ```
  </Accordion>
  
  <Accordion title="Routes return 404 after deployment">
    Verify `firebase.json` includes the SPA rewrite:
    
    ```json
    "rewrites": [
      {
        "source": "**",
        "destination": "/index.html"
      }
    ]
    ```
  </Accordion>
  
  <Accordion title="Firestore quota exceeded">
    Monitor usage in Firebase Console
    
    Upgrade to Blaze plan for higher limits:
    - Spark (free): 50K reads, 20K writes per day
    - Blaze: Pay-as-you-go pricing
  </Accordion>
  
  <Accordion title="Authentication issues in production">
    Verify authorized domains in Firebase Console:
    
    **Authentication** → **Settings** → **Authorized domains**
    
    Add your custom domain if using one
  </Accordion>
</AccordionGroup>
